<P align=justify>Для <STRONG>Настройки</STRONG> указывается:</P>
<UL>
<LI>
<DIV align=justify>наименование</DIV>
<LI>
<DIV align=justify>дата начала ее действия</DIV>
<LI>
<DIV align=justify>признаки отражения в учетах: УУ, БУ, НУ</DIV>
<LI>
<DIV align=justify>вариант: общая, УСН (доходы), УСН (доходы - расходы)</DIV></LI></UL>
<P align=justify>Признаки отражения в учетах устанавливают ограничение, по каким видам учета может быть проведена процедура закрытия месяца, в которой используется эта настройка. 
<P align=justify>Вариант настройки определяет, для какой учетной политики по&nbsp;регл. учету &nbsp;предназначена эта настройка. 
<HR>
</SPAN><SPAN id=СводныеНастройки_Показывать class=top>На закладке <STRONG><FONT color=#0054e3>Сводные настройки</FONT></STRONG> отображен состав регламентных операций. 
<P></P>
<P>Для каждой регламентной операции&nbsp;можно:</P>
<UL>
<LI>включить/отключить ее использование 
<LI>назначить ответственного за ее&nbsp;выполнение</LI></UL>
<P><STRONG>См.также</STRONG></P>
<P>На закладке <A id=Команда href="1c:ОткрытьЗакладкуСхема">Схема</A> в графическом виде отображается состав и последовательность выполнения регламентных операций.</P>
<P>На закладке <A id=Команда href="1c:ОткрытьЗакладкуСхемаНДС">Схема расчета НДС</A> в графическом виде отображается детализированный состав и последовательность выполнения регламентных операций по расчету НДС. <BR>На общей схеме операция по расчету НДС представлена в виде общего блока "Рассчитать НДС".</P>
<P>На закладке&nbsp;<A id=Команда href="1c:ОткрытьЗакладкуОтветственные">Ответственные</A>&nbsp;удобно назначать ответственных за выполнение регламентных операций.</P>
<P>На закладке <A id=Команда href="1c:ОткрытьЗакладкуРаспределениеЗатрат">Распределение затрат</A> назначаются способы распределения затрат для расчета себестоимости.</P></SPAN><SPAN id=Схема_Показывать class=top>На&nbsp;закладке <STRONG><FONT color=#0054e3>Схема</FONT></STRONG> отображается состав и последовательность выполнения регламентных операций. 
<P>Здесь&nbsp;можно:</P>
<UL>
<LI>Отключить/включить регламентную операцию&nbsp;(двойной щелчок мыши на операции) 
<LI>Назначить ответственного (контекстное меню&nbsp;операции при нажатии правой клавиши мыши)</LI></UL></SPAN><SPAN id=СхемаНДС_Показывать class=top>На&nbsp;закладке <STRONG><FONT color=#0054e3>Схема расчета НДС</FONT></STRONG> отображается детализированный состав и последовательность выполнения регламентных операций по расчету НДС. <BR>На общей схеме операция по расчету НДС представлена в виде общего блока "Рассчитать НДС". 
<P>Здесь&nbsp;можно:</P>
<UL>
<LI>Отключить/включить регламентную операцию&nbsp;(двойной щелчок мыши на операции) 
<LI>Назначить ответственного (контекстное меню&nbsp;операции при нажатии правой клавиши мыши)</LI></UL></SPAN><SPAN id=РаспределениеЗатрат_Показывать class=top>На&nbsp;закладке <STRONG><FONT color=#0054e3>Распределение затрат</FONT></STRONG> назначаются способы распределения затрат, которые&nbsp;будут применены&nbsp;при расчете себестоимости. 
<P></P>
<P>Способы распределения затрат можно назначать с точностью до характера затрат, подразделений, статей затрат и номенклатурных групп. При этом настройку можно выполнить с меньшей детализацией, например:</P>
<UL>
<LI>для конкретного <STRONG>вида</STRONG> <STRONG>подразделения</STRONG>&nbsp; и/или конкретного <STRONG>характера затрат</STRONG> 
<LI>для конкретного <STRONG>подразделения</STRONG> и/или <STRONG>статьи затрат</STRONG> 
<LI><STRONG>для всех</STRONG> подразделений и/или&nbsp;статей затрат 
<LI>и т.д.</LI></UL>
<P>При определении подходящего способа распределения затрат порядок приоритетов (по убыванию значимости) следующий:</P>
<UL>
<LI>Характер затрат 
<LI>Вид подразделения 
<LI>Подразделение 
<LI>Статья затрат 
<LI>Номенклатурная группа</LI></UL>
<P><EM><STRONG>Например</STRONG>, список содержит две строки:</EM></P>
<P><EM>1. «Вид подразделения» – <STRONG>по всем видам подразделений</STRONG>, «Статья затрат» - <STRONG>оплата труда</STRONG></EM></P>
<P><EM>2. «Вид подразделения» - <STRONG>основное производство</STRONG>, «Статья затрат» – <STRONG>по всем статьям затрат</STRONG></EM></P>
<P><EM>Для расходов&nbsp;<STRONG>основного производства</STRONG> способ распределения затрат будет определяться по второй строке, т.к. в ней&nbsp;<STRONG>вид подразделения</STRONG> указан <STRONG>явно</STRONG>. </EM></P>
<P>Настройки способов распределения затрат - общие для всех учетов: и для управленческого, и для регламентированного (БУ и НУ). <BR>При этом,&nbsp;в регламентированном учете настройки действуют для <STRONG>подразделений организаций</STRONG>, которые соответствуют указанному <STRONG>подразделению</STRONG></P>
<H3>Особенности настройки распределения отдельных затрат</H3>
<P>Распределение <STRONG>расходов на продажу</STRONG> (коммерческих расходов) целесообразно настраивать только для целей регламентированного учета.<BR>Если настройка <STRONG>не выполнена</STRONG>, то коммерческие расходы будут 
<UL>
<LI>списаны на счет 90.07 
<LI>распределены между видами деятельности пропорционально оборотам счета 90.01 </LI></UL>
</P>
<P>Если используется <STRONG>директ-костинг</STRONG>, то распределение <STRONG>общехозяйственных расходов</STRONG> (управленческих расходов) целесообразно настраивать только для целей регламентированного учета.<BR>Если настройка <STRONG>не выполнена</STRONG>, то управленческие расходы будут 
<UL>
<LI>списаны на счет, указанный в учетной политике 
<LI>распределены между видами деятельности пропорционально оборотам счета 90.01 </LI></UL>
</P>
<H3>Особенности выполнения и использования настройки, содержащей отдельные способы распределения</H3>
<P>Следует учитывать некоторые особенности выполнения и использования настройки, в которой</P>
<UL>
<LI>либо задан хотя бы один из способов распределения 
<UL>
<LI>Директ-костинг 
<LI>Перенести на производственные расходы 
<LI>По выручке от реализации услуг 
<LI>Расходы на продажу 
<LI>с типом базы «Табличная» 
<LI>с типом базы «По данным ИБ» </LI></UL>
<LI>либо указаны номенклатурные группы</LI></UL>
<P>А именно:</P>
<OL>
<LI>
<P>При выполнении настройки <STRONG>характер затрат</STRONG>&nbsp;следует указать <STRONG>явно</STRONG></P>
<LI>
<P>Настройку следует использовать только для <STRONG>регламентированного</STRONG> учета.</P>
<P><EM>Следствие: для управленческого учета следует использовать <STRONG>отдельную</STRONG> настройку закрытия месяца, в ней новые способы распределения <STRONG>не указывать</STRONG> </P></EM>
<LI>
<P>Отчет «Калькуляция себестоимости» содержит только суммы затрат (не содержит количества затрат)</P>
<LI>
<P>Услуги собственным подразделениям и полуфабрикаты, переданные в производство в том же месяце, в котором выпущены, в налоговом учете считаются прямыми расходами</P>
<LI>
<P>Затраты, распределение которых описано настройкой закрытия месяца, не следует распределять до расчета себестоимости документами:<UL>
<LI>"Акт об оказании производственных услуг"
<LI>"Отчет производства за смену"
<LI>"Поступление товаров из переработки"
<LI>"Распределение материалов на выпуск"
<LI>"Распределение прочих затрат"
</P></LI></UL></LI>
</OL>
<P></P></SPAN><SPAN id=Ответственные_Показывать class=top>На закладке <STRONG><FONT color=#0054e3>Ответственные</FONT></STRONG> назначаются ответственные (пользователь&nbsp;или группа пользователей) за выполнение каждой регламентной операции. 
<P>Ответственного можно назначить одним из способов:</P>
<UL>
<LI>указать непосредственно в списке 
<LI>выбрать&nbsp;двойным щелчком мыши на элементе списка пользователей (групп) 
<LI>«перетащить» из списка пользователей (групп) 
<LI>использовать специальную кнопку <STRONG>&lt;</STRONG></LI></UL>
<P>С помощью кнопки&nbsp;<STRONG>&lt;</STRONG>&nbsp;можно одновременно назначить ответственного сразу за&nbsp;несколько операций:</P>
<OL>
<LI>в левом табличном поле выбрать несколько регламентных операций (отмечать, удерживая клавишу Ctrl или Shift) 
<LI>в правом табличном поле выбрать пользователя (группу) 
<LI>нажать кнопку <STRONG>&lt;</STRONG></LI></OL></SPAN><SPAN id=КолонкиВидимы_Показывать class=top>
<P><STRONG><FONT color=#ff0000>Важно!</FONT></STRONG> Принадлежность регламентной операции &nbsp;к видам учета (УУ, БУ, НУ)&nbsp;устанавливается автоматически при создании настройки и отображается для справки. Для изменения <STRONG>недоступна</STRONG>.</P></SPAN></BODY></HTML>
