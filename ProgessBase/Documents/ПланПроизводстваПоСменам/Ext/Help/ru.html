Документ «План производства по сменам»

Документ предназначен для отражения в управленческом учете следующих операций:

  • Ввод в систему и хранение данных по структуре и объемам цехового производства
  • Формирование календарного плана производства и планов производства и потребления по сменам

Ввод в систему и хранение данных по структуре и объемам цехового производства

Идеология данного документа такова, что он не является исключительно регистрирующим. В данном документе производится ввод источников данных по планируемому производству и расчет производственной программы, которая удовлетворяет требованиям, наложенным источниками. Среди таких источников могут быть уточненные планы производства, заказы покупателей и индивидуально указанные выпуски продукции.

Рассчитываемая производственная программа формируется при заполнении документа по данным источников по кнопке «Выполнить планирование».

В документе присутствуют реквизиты «Подразделение» и «Проект». Эти реквизиты являются справочными и заполняются для выделения специфичных документов.

Исходные данные вносятся в три табличных части. При формировании посменного плана производства учитываются данные из всех заполненных табличных частей.

Табличная часть – «Производство по планам». В нее вносится перечень уточненных планов производства, по составу которых будет производиться посменное планирование производства. В каждой строке этого перечня указываются:

  • сценарий, определяющий, какие данные следует извлекать из уточненных планов производства
  • подразделение и проект, дополнительно уточняющие выборку
  • интервал дат, в котором эти данные должны извлекаться
  • дата запуска, определяет, в который день должен быть произведен первый выпуск по данной строке
  • дата выпуска, определяет, в который день должен быть произведен последний выпуск по данной строке
  • вариант распределения, определяет, как будут распределены частные выпуски внутри интервала.

Табличная часть – «Производство по заказам». В нее вносится перечень заказов, по которым производится посменное планирование производства. В каждой строке указывается:

  • заказ покупателя, неисполненные обязательства по которому будут входить в производственную программу;
  • проект, к которому будут привязаны выпуски;
  • дата запуска, определяет, в который день должен быть произведен первый выпуск по данной строке;
  • дата выпуска, определяет, в который день должен быть произведен последний выпуск по данной строке;
  • вариант распределения, определяет, как будут распределены частные выпуски внутри интервала.

Табличная часть – «Индивидуальные выпуски». В нее вносится явный перечень выпускаемой номенклатуры, которую нужно произвести. В каждой строке указывается:

  • номенклатура и характеристика номенклатуры, производство которой следует распланировать;
  • единица измерения и количество номенклатуры;
  • спецификация планирования, по которой следует производить номенклатуру;
  • заказ, к которому будут привязаны выпуски;
  • проект, к которому будут привязаны выпуски;
  • дата запуска, определяет, в который день должен быть произведен первый выпуск по данной строке;
  • дата выпуска, определяет, в который день должен быть произведен последний выпуск по данной строке;
  • вариант распределения, определяет, как будут распределены частные выпуски внутри интервала;

Табличная часть «Индивидуальные выпуски» может заполняться штатным подбором из справочника «Номенклатура».

Сформированные данные календарного плана производства и потребления по сменам содержатся в двух табличных частях.

Табличная часть – «Занятость рабочих центров». Содержит сформированный при заполнении календарный план производства.

Табличная часть – «Потребности производства». Содержит сформированные при заполнении потребности производства.

Заполнение варианта распределения, даты запуска и даты выпуска определяется следующими факторами:

  1. Если номенклатура по данной строке (заказу, сценарию) должна быть произведена к строго установленному сроку, вариант распределения следует установить в значение «к последней дате». В этом случае, при заполнении документа расчетный модуль назначит сменные здания таким образом, чтобы выпуск продукции пришелся как можно ближе к дате выпуска. Дата запуска при этом игнорируется.
  2. Если номенклатура по данной строке (заказу, сценарию) должна быть произведена условно-равномерно в некотором интервале дат, вариант распределения следует установить в значение «равномерно на период планирования». В этом случае, при заполнении документа расчетный модуль сначала сформирует календарный план производства, в котором выпуски конечной (целевой) продукции по данной строке (заказу, сценарию) будут распределены в указанном интервале пропорционально. Затем для каждого из составляющих выпусков будет произведено собственное посменное планирование.

Первый вариант более приемлем для производства специфичной номенклатуры под заказ, второй – для равномерного производства по планам.

После заполнения данных по источникам следует указать также параметры планирования на закладке:

  • перечень доступных складов, то есть мест хранения, ТМЦ с которых могут использоваться расчетным механизмом как доступные к использованию;
  • момент запасов. Это момент времени, на который будет извлекаться информация о доступности запасов;
  • начальная граница. Временной момент, определяющий границу, ранее которой данные посменного планирования не могут быть скорректированы.

При определении начальной границы критерием может выступать момент (дата) по которую невозможно произвести замену сменно-суточных заданий в производственных участках.

Формирование календарного плана производства и планов производства и потребления по сменам

Формирование производится в ходе заполнения документа автоматически по данным, указанным пользователем в документе. Сформированный календарный план производства сохраняется в информационной базе и может быть изменен.

Для записанного документа возможно формирование печатных форм: календарного плана производства и собственно печатной формы состава документа.

Документ не используется для ведения регламентированного учета.
